## 1. 환경 준비

실행:

```sh
cd /Users/jaehojoo/Desktop/codex-lgcns/real-sattie-study
./one-shot-setup.sh
```

- `venv` 디렉터리가 존재한다.

## 7. API 엔드포인트 검증

실행:

```sh
curl http://127.0.0.1:6003/
curl http://127.0.0.1:6003/health
curl http://127.0.0.1:6003/requests
curl http://127.0.0.1:6003/requests/REQ-20260307-SEOUL-001
curl http://127.0.0.1:6003/requests/REQ-20260307-WESTSEA-SAR-001
curl http://127.0.0.1:6003/requests/REQ-20260307-SEOUL-001/request-candidates
curl http://127.0.0.1:6003/requests/REQ-20260307-SEOUL-001/request-candidates/OPT-CAND-001
curl -X POST http://127.0.0.1:6003/requests/REQ-20260307-SEOUL-001/simulate-candidate-input -H 'content-type: application/json' -d '{"sensor_type":"SAR","priority_tier":"URGENT","area_km2":50,"window_hours":168,"cloud_pct":18,"max_cloud_pct":20,"required_off_nadir_deg":18.5,"max_off_nadir_deg":25,"predicted_incidence_deg":30,"min_incidence_deg":25,"max_incidence_deg":40,"sun_elevation_deg":44,"min_sun_elevation_deg":20,"coverage_ratio_predicted":0.98,"coverage_ratio_required":0.95,"expected_data_volume_gbit":14,"recorder_free_gbit":48,"recorder_backlog_gbit":12,"available_downlink_gbit":42,"power_margin_pct":19,"thermal_margin_pct":16}'
curl -X POST http://127.0.0.1:6003/requests/REQ-20260307-SEOUL-001/request-candidates/OPT-CAND-001/simulate
curl -X POST http://127.0.0.1:6003/simulate -H 'content-type: application/json' -d '{"sensor_type":"OPTICAL","priority_tier":"STANDARD","area_km2":400,"window_hours":168,"cloud_pct":34,"max_cloud_pct":20,"required_off_nadir_deg":20.5,"max_off_nadir_deg":25,"predicted_incidence_deg":30,"min_incidence_deg":25,"max_incidence_deg":40,"sun_elevation_deg":42,"min_sun_elevation_deg":20,"coverage_ratio_predicted":0.97,"coverage_ratio_required":0.95,"expected_data_volume_gbit":14,"recorder_free_gbit":48,"recorder_backlog_gbit":10,"available_downlink_gbit":40,"power_margin_pct":19,"thermal_margin_pct":15}'
curl http://127.0.0.1:6003/docs
```

확인할 것:

- `/`가 프론트엔드 HTML을 반환한다.
- `/health`가 `{"status":"ok"}`를 반환한다.
- `/requests`가 요청 2건을 반환한다.
- `POST /requests`가 새 내부 요청을 생성하고 내부 요청코드를 자동 발번한다.
- optical 상세 응답에 `CONDITIONALLY_FEASIBLE`가 보인다.
- SAR 상세 응답에 `CONDITIONALLY_FEASIBLE`가 보인다.
- `/requests/{request_code}/external-refs`가 외부 요청번호 매핑 목록을 반환한다.
- `PATCH /requests/{request_code}/external-refs/{id}`가 primary 외부 참조를 변경한다.
- `DELETE /requests/{request_code}/external-refs/{id}`가 외부 요청번호 매핑을 삭제하고 필요하면 다른 참조를 primary로 승격한다.
- `/requests/{request_code}/request-candidates`가 후보 목록을 반환한다.
- `/requests/{request_code}/request-candidates/{candidate_code}`가 후보 상세를 반환한다.
- `/requests/{request_code}/simulate-candidate-input`이 request context를 붙인 현재 입력 기준 평가를 반환한다.
- 정책 위반 입력에는 `POLICY` stage reason이 포함된다.
- SAR 편파 불일치 상황에서는 `POLARIZATION_UNSUPPORTED`가 POLICY stage reason으로 반환된다.
- 요청 전체 `overall_probability`가 best candidate 단일 확률이 아니라 누적 성공확률로 보이는지 확인한다.
- 요청 전체 응답에 `first_feasible_attempt_at`, `expected_attempt_count`가 포함되는지 확인한다.
- 요청 전체 응답에 `required_attempt_count`, `repeat_requirement_met`, `repeat_quality_attempt_count`, `repeat_spacing_hours_required`, `repeat_spacing_met`, `repeat_incidence_tolerance_deg`, `repeat_incidence_consistent_count`, `repeat_incidence_met`가 포함되는지 확인한다.
- 요청 전체 응답에 `repeat_quality_threshold`가 포함되는지 확인한다.
- 요청 전체 응답에 `proposal` 객체가 포함되는지 확인한다.
- 초기 후보 상세 응답의 `latest_run`은 비어 있거나 `null`이다.
- `POST /requests/{request_code}/request-candidates/{candidate_code}/simulate`가 최신 후보 실행 결과를 저장한다.
- `/simulate`가 synthetic verdict JSON을 반환한다.
- `/docs`가 FastAPI Swagger UI HTML을 반환한다.

## 8. 프론트엔드 수동 검증

브라우저에서 아래 주소로 접속한다.

```text
http://127.0.0.1:6003/
```

확인할 것:

- 좌측 요청 목록에 2개 요청이 보인다.
- 좌측 `새 요청 생성` 폼에서 신규 요청을 만들 수 있다.
- `외부 요청번호 매핑` 패널은 기본 닫힘 상태이고, 토글 버튼으로 열고 닫을 수 있다.
- `서울 AOI 광학 촬영 요청건`을 선택하면 `REQ-20260307-SEOUL-001`이 로딩된다.
- 요청 하위 후보건 3개가 보인다.
- 광학 후보를 선택하면 입력값이 로드되고, 상단 요청 개요/운용 제한은 고객 원본 요청값으로 읽기 전용이다.
- 초기 상태에서는 `현재 입력 기준 평가`는 보이지만 `최신 저장 실행 결과`는 `미실행`이다.
- 요청 전체 섹션은 현재 후보 입력값 기준으로 계산된 동적 평가를 보여준다.
- `Feasibility Analysis Summary` 패널에 누적 성공확률, 첫 촬영기회, 예상 촬영기회 소진 수, SLA 요약이 표시된다.
- 후보 입력값을 수정하면 `현재 입력 기준 평가`가 자동으로 다시 계산된다.
- SAR 요청을 선택하면 `REQ-20260307-WESTSEA-SAR-001`이 로딩된다.
- SAR 후보 3개가 보인다.
- 후보 목록에는 `현재 평가`와 `저장 실행` 배지가 함께 보인다.
- 새 후보 생성 시 후보코드는 입력하지 않아도 저장 후 자동 생성된다.
- 외부 요청번호는 별도 매핑이며 `POST /requests/{request_code}/external-refs`로 추가 연결할 수 있다.
- 외부 요청번호 목록에서 `기본 지정`, `삭제` 버튼이 동작한다.
- `새 후보 준비`가 신규 후보 초안을 연다.
- `후보 저장`이 정상 동작한다.
- `저장 후 검증 실행` 후 `최신 저장 실행 결과`의 verdict와 reasons가 새로 계산되어 갱신된다.
- `기준안 여부`를 체크해 저장하면 같은 요청의 기존 기준안이 자동 해제되는지 확인한다.
- 요청을 다시 열면 기준안 후보가 먼저 로드되는지 확인한다.
- `incidence_window` proposal 카드의 `기존 후보 반영 후 실행` 버튼을 누르면 대상 후보 입력값이 즉시 갱신되고 바로 검증 실행된다.
- `candidate_split` proposal 카드의 `분리 후보 생성` 버튼을 누르면 새 `DRAFT` 후보가 생성되고 즉시 검증 실행까지 수행된다.
- 후보 상세의 `실행 이력`에 `input vN`과 실행 출처가 남는지 확인한다.
- `실행 이력`의 `전체 이력` / `추천안 실행만` 필터가 기대대로 동작하는지 확인한다.
- recommendation에 `expected_probability_gain`이 보이는지 확인한다.
- `선택 후보 삭제`가 동작한다.
